Browser Fingerprinting

Discover how browsers collect unique data to create a digital fingerprint for tracking and identification.

Narration

Context: Browser fingerprinting is a growing concern in the digital age, where websites can track users without their explicit consent. This presentation introduces the concept and its mechanics.

Information: We’ll explore how browsers gather data from various sources, like hardware and software settings, to create a unique identifier. This process is invisible to most users but has significant privacy implications.

Key Learning Point: Browser fingerprinting enables tracking without cookies, making it a powerful and stealthy tool for identifying users across the web.

What is Browser Fingerprinting?

A technique to identify users by collecting unique browser and device characteristics, creating a distinctive "fingerprint" without cookies.

  • Uses APIs like navigator, screen, and window.
  • Generates a hash for comparison.
  • Downloadable as JSON for analysis.

Narration

Context: Unlike cookies, which users can clear, fingerprinting relies on inherent device and browser traits that are harder to modify.

Information: The technique collects data through JavaScript APIs, such as navigator for browser details and screen for display settings. A hash is generated to summarize the fingerprint, and the data can be exported as JSON for further study.

Key Learning Point: Fingerprinting creates a unique user profile by combining multiple browser and device attributes, bypassing traditional tracking defenses.

Fingerprinting Process

Collects attributes in two modes:

  • With Timestamp: Includes timestamp (gatherFingerprint()).
  • No Timestamp: Excludes timestamp for consistent hashing (gatherFingerprintNoTimestamp()).
  • Hashes data with simpleHash().
  • Handles errors, displays or downloads JSON.

Narration

Context: The fingerprinting tool is designed to be robust, offering flexibility in how data is collected and presented.

Information: Two functions collect attributes: one includes a timestamp for temporal context, while the other excludes it for stable hashes. Errors are managed with try-catch blocks, ensuring inaccessible data is flagged. The resulting data is hashed for quick comparison and can be viewed or downloaded.

Key Learning Point: The fingerprinting process is systematic, error-resistant, and adaptable, allowing consistent tracking with or without timestamps.

Timestamp

Records generation time

Process: Uses new Date().toISOString().

Purpose: Adds temporal context, but affects hash consistency.

Note: Only in "With Timestamp" mode.

Narration

Context: Timestamps are common in data collection but pose challenges for fingerprinting due to their dynamic nature.

Information: The timestamp is captured using new Date().toISOString(), providing a precise record of when the fingerprint was generated. However, its inclusion changes the hash each time, making the "No Timestamp" mode preferable for consistent tracking.

Key Learning Point: Timestamps provide useful context but reduce fingerprint stability, requiring a choice between temporal accuracy and hash consistency.

User Agent

Browser and OS identifier

Process: Accessed via navigator.userAgent.

Purpose: Distinguishes browsers and OS (e.g., Chrome on Windows).

Narration

Context: The user agent is one of the most accessible pieces of browser information, often used in web development and tracking.

Information: Retrieved via navigator.userAgent, it reveals the browser type, version, and operating system. For example, it might show "Mozilla/5.0 (Windows NT 10.0; Win64; x64) Chrome/120.0.0.0". This helps differentiate devices and browsers.

Key Learning Point: The user agent is a foundational fingerprinting attribute, providing broad but essential identification details.

Key Learning Point: App version adds minor specificity to fingerprinting, though its overlap with user agent limits its standalone value.
